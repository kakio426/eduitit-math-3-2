# 개인정보와 운영 메모

## 저장하지 않는 정보

- 학생 실명
- 학교명
- 지역
- 학년/반 실명 라벨
- 전화번호
- 이메일
- 정확한 위치

## 저장하는 정보

- 서버 생성 닉네임
- 선택적 참가코드 해시
- 차시 ID
- 점수
- 정답 수
- 플레이 시간
- 문제별 answer log
- 서버 검산 상태

## 공개 랭킹 표시 규칙

전국 주간 랭킹에는 아래만 표시합니다.

- 순위
- 서버 생성 닉네임
- 점수
- 정답 수
- 차시
- 주차 시작일

전국 주간 랭킹에는 참가코드, 학교명, 지역, 세션 ID, DB ID, IP 주소를 넣지 않습니다.

## 닉네임 정책

학생이 직접 닉네임을 쓰지 않습니다. 서버가 안전 단어 목록과 숫자를 조합합니다.

예:

- `반짝 매스몬 27`
- `튼튼 로켓 43`
- `초록 점프 83`

서버가 만든 닉네임과 다르면 점수 제출을 거절합니다.

## 참가코드 정책

참가코드는 선택값입니다. 쓰는 경우 `M7K2Q` 같은 랜덤 문자열만 허용합니다.

금지 예:

- 학교명
- 지역명
- 교사명
- 실제 반 이름

DB에는 참가코드 원문 대신 해시를 저장합니다.

## 보관 기간

기본 권장안:

- 원본 answer log: 90일
- 주간 집계 랭킹: 운영 정책에 따라 더 길게 보관 가능

운영 전 업체는 삭제 요청, 숨김 처리, 백업 보관 기간을 확정해야 합니다.

## 관리자 기능 책임

MVP 코드는 관리자 API의 운영 요구사항을 문서화합니다. 공개 운영 전 업체가 다음을 확정해야 합니다.

- 관리자 인증 방식
- 관리자 계정 소유자
- 점수 숨김/삭제 권한
- 감사 로그 보관 기간
- 이상 점수 신고/처리 절차

처음 조치는 hard delete보다 soft hide를 권장합니다.

## 공식 안내서 확인

공개 운영 전 업체는 최신 개인정보보호위원회 아동·청소년 개인정보 보호 안내서를 확인하고, 개인정보 처리방침과 서비스 화면에 필요한 고지를 반영해야 합니다.
