# 매스몬 0 공장 설명 보고서

## 1. 개요

`매스몬 0 공장`은 3학년 2학기 1단원 3차시에서 다루는 `(몇십)×(몇십), (몇십몇)×(몇십)` 계산을 짧은 게임 흐름으로 연습하는 수학 게임입니다. 학생은 곱셈 문제를 보고 0을 잠깐 떼어 낸 곱을 먼저 고른 뒤, 끝에 다시 붙일 0의 개수를 골라 답을 만듭니다.

핵심 목표는 `0 없는 곱셈을 먼저 한 뒤, 두 수 끝의 0 개수를 합쳐 답 뒤에 붙이는 원리`를 자연스럽게 반복하게 만드는 것입니다.

## 2. 학습 설계

- 문제 유형: `(A0)×(B0)`, `(AB)×(C0)`
- 문제 은행: 두 유형에서 매 판 5문제씩 섞어 총 10문제 출제
- 입력 방식: 1단계 0 뗀 곱 선택, 2단계 0 개수 선택
- 피드백: 첫 오답에는 짧은 힌트, 두 번째 오답에는 정답과 설명을 보여 준 뒤 다음 단계로 이동
- 대표 오개념 처리: 2단계 선택지에는 0을 한 개 적게 붙이거나 한 개 많이 붙이는 보기가 항상 들어감
- 보상: 한 문제 완료마다 상자 칸 1칸과 공장 힘 이벤트 1회 적용. 정답 문제는 상자 출발, 상자 3개 출발, 상자 고치기, 문 열림, 컨베이어 멈춤, 무지개 상자 중 하나가 나오고, 오답이 있었던 문제는 고칠 상자로 표시. 보상은 컨베이어에서 이동하는 닫힌 장난감 상자와 짧은 모달로 보여 줌
- 상자가 간 곳: 공장 힘 1~100을 확인해 우리 반 -> 학교 -> 동네 -> 도시 -> 전국 -> 세계를 공개
- 특별 단계: 무지개 상자를 한 번 얻으면 일반 공장 힘보다 우주가 우선
- 최종 보상: 상자가 간 곳과 정답 수 조건을 함께 보아 0 공장 매스몬 카드 1장을 공개하고, 카드 다운로드를 생성함

## 3. 게임 흐름

```text
첫 화면 -> 설명 화면 -> 0 뗀 곱 선택 -> 0 개수 선택 -> 0 이동 -> 장난감 상자 출발 -> 다음 문제 또는 바로 결과 -> 공장 힘 확인 -> 상자가 간 곳 + 매스몬 카드
```

학생은 예를 들어 `30 × 40`에서 먼저 `3 × 4 = 12`를 고릅니다. 다음 단계에서 두 수 끝에 있던 0이 모두 2개였다는 것을 골라, 컨베이어 보드에 0 두 개가 붙고 `1,200`이 됩니다.

`27 × 40`에서는 `27 × 4 = 108`을 고른 뒤 0을 1개 붙여 `1,080`을 만듭니다. 이처럼 문제마다 통째 계산보다 `0 뗀 곱`과 `0 개수`가 먼저 보이도록 설계했습니다.

## 4. 화면별 설명

### 첫 화면

첫 화면은 `cover-generated.webp`를 RasterStage 배경으로 사용합니다. 0 공장과 컨베이어가 보이는 대표 장면 위에 GPT Image로 만든 독립 제목 로고(`title-logo-generated.webp`)를 얹고, 한 줄 목표와 시작 버튼은 HTML로 얹습니다. 제목 로고 원본은 `title-logo-chromakey.png`로 보관하고, 초록 배경 제거 PNG/WebP를 배포 자산으로 사용합니다.

### 설명 화면

설명 화면은 `tutorial-zero-flow-generated.webp`를 RasterStage 배경으로 사용합니다. 생성 이미지 안에는 한글이나 숫자를 굽지 않고, 0이 떨어졌다가 다시 붙고 닫힌 장난감 상자가 움직이는 흐름만 보여 줍니다. 정확한 설명과 예시는 HTML 오버레이로 얹어 철자 오류와 잘못된 수식이 화면에 남지 않게 했습니다.

설명 문구는 3단계만 보여 줍니다.

1. 끝의 0을 잠깐 떼고 0 없는 곱셈을 계산합니다.
2. 떼어 낸 0의 개수를 세어 다시 붙입니다.
3. 완성한 상자를 컨베이어로 보냅니다.

버튼 문구는 다음 행동이 바로 보이도록 `문제 시작`으로 둡니다.

### 문제 화면

문제 화면은 위쪽에 큰 문제·현재 단계·한 줄 지시·선택지를 두고, 아래쪽에 컨베이어 보드와 상자 칸을 둡니다. 컨베이어 보드는 `0 뗀 곱`, `붙일 0`, `답` 세 칸으로 구성됩니다. 1단계를 맞히면 0 뗀 곱 칸이 채워지고, 2단계를 맞히면 0이 실제 개수만큼 나타난 뒤 답이 자동으로 채워집니다.

상자 칸은 문제마다 1칸씩 채워집니다. 한 번에 맞힌 문제는 노란 완성 상자로 남고, 중간에 틀린 문제는 빨간 고칠 상자로 남아 학생이 자신의 진행을 숫자만이 아니라 공장 상태로 볼 수 있게 했습니다.

### 보상 화면

한 문제의 두 단계가 끝나면 0이 답 칸으로 이동하고, 이어서 닫힌 장난감 상자가 컨베이어 위를 지나갑니다. 보통은 상자 1개가 출발하고, 대량 이벤트는 상자 3개, 고치기 이벤트는 빨간 오버레이, 멈춤 이벤트는 정지 오버레이, 무지개 이벤트는 무지개 오버레이로 보입니다. 문제 안에서 한 번이라도 틀리면 `고칠 상자`가 적용됩니다.

보상 모달은 설명 문장을 없애고 닫힌 장난감 상자 이미지와 한 줄 라벨만 보여 줍니다. 버튼은 다음 행동만 말하도록 `다음` 또는 `보기`로 둡니다. 내부 계산은 유지하지만 학생 화면에는 `%` 변화량을 보상 모달에 보이지 않게 했습니다.

### 결과 화면

결과 화면은 공장 장면 RasterStage 배경 위에 공장 힘 막대, 상자가 간 곳 트랙, 정답 수, 매스몬 카드를 HTML로 얹습니다. 공장 힘은 상자가 간 곳을 정하고, 무지개 상자는 우주 결과를 엽니다. 매스몬 카드는 공장 힘뿐 아니라 정답 수 조건도 함께 보아 공개됩니다. 공장 힘이 0이면 별도의 컨베이어 점검 배경을 사용해 성공 결과와 재도전 결과가 시각적으로 구분됩니다.

## 5. 매스몬 역할

3차시의 매스몬은 1차시 기준으로 새로 만든 `0 공장 동물 매스몬팩` 10종 중 이번 판 결과에 맞는 카드 1장으로 공개됩니다. 0 공장의 중심 보상은 공장 힘과 상자가 간 곳이고, 매스몬 카드는 그 결과를 기념하는 최종 보상입니다. 중간 보상에서는 매스몬을 미리 보여 주지 않고, 닫힌 장난감 상자만 보여 줍니다.

현재 기준은 1차시 `매스몬 상자런`의 밝은 2D 애니/스티커형 매스몬 톤입니다. 3차시 전용 팩은 톱니바퀴나 컨베이어 같은 사물형 매스몬이 아니라, 햄스터·토끼·수달·고양이·다람쥐·펭귄·거북이·판다·부엉이·별사슴처럼 동물/판타지 생물을 본체로 삼습니다. 0 공장 느낌은 작업모, 0 스티커, 배지 같은 소품으로만 더했습니다. 원본 관리는 `_shared/mathmon/zero-factory-animal-pack/`에서 하고, 3차시 실행 패키지에는 `assets/mathmon/zero-factory-animal-pack/*.webp` 배포본을 복사했습니다. 기존 0 공장팩과 V2 장난감/클레이풍 팩은 보존만 하고 실행에는 쓰지 않습니다.

장난감 상자 보상 자산은 `_shared/mathmon/zero-factory-animal-pack/toybox/`에서 관리합니다. 실행 기준 구성은 닫힌 장난감 상자 1장, 하단 컨베이어 무대 2장, 이벤트 오버레이 3장입니다. 캐릭터가 보이는 초기 toybox 시안은 보존만 하고 실행 패키지에는 복사하지 않습니다.
